Instrukcja do gita

Co z tym gitem?:)

Git to system kontroli wersji przydatny przy tworzeniu projektów przez jedną lub więcej osób. Zapamiętuje całą historię zmian w plikach projektu, które przechowywane są w repozytoriach. Projekt można udostępniać innym osobom (do wglądu lub z możliwością dopisywania/modyfikowania kodu).

Jak zacząć?

Najczystsza wersja gita to program git uruchamiany z wiersza poleceń. Można też używać SourceTree lub innych programów, które mają okienkowy interfejs i dzięki temu są łatwiejsze w użyciu.

Tworzenie nowego projektu

  1. Aby utworzyć na GitHubie nowy projekt, trzeba się zalogować do swojego konta i utworzyć repozytorium przyciskiem New Repository. Repozytorium to taka piaskownica, która na początku jest pusta.
  2. Teraz robimy klona za pomocą narzędzia do gita. Na GitHubie na stronie Twojego repozytorium znajdziesz jego adres (clone url).
  3. Otwieramy SourceTree (lub inne narzędzie): File, New/Clone; wyświetla się Bookmarks; wklejamy adres.
  4. Otwieramy edytor (np. Sublime, Brackets) i pracujemy w projekcie.
  5. Kiedy jakiś fragment pracy jest ukończony, to wtedy pora zrobić commit, czyli zatwierdzić zmiany. W momencie commitu, robisz "zdjęcie" zawartości wszystkich plików projektu i dzięki temu pracując dalej można zawsze przywrócić projekt do dowolnego commitu, zwłaszcza, gdy ktoś coś popsuje. Otwieramy narzędzie do gita (SourceTree), zaznaczamy "uncommitted changes" i klikamy "Commit". Zmiany zostają zapamiętane, ale tylko na Twoim komputerze, więc żeby inni mogli zobaczyć wprowadzone zmiany, musisz kliknąć "Push".
  6. I potem znów powrót do edytora (Sublime, Brackets) i pętla się zamyka.

Git - polecenia

  1. git config

    Polecenie używane jednorazowo przy konfiguracji instalacji gita użytkownika:

    git config --global user.name "John Doe"

    git config --global user.email "John.Doe@example.com"

  2. git init

    3. Zakłada nowe repozytorium w bieżącym katalogu i od tej pory można robić commity

  3. git add

    4. Dodaje nowe pliki do repozytorium

    git add index.html

    Dodaje wskazany plik.

    git add .

    Dodaje wszystkie pliki będące w zasięgu repozytorium

  4. git status

    To polecenie wyświetli aktualny status pracy nad projektem. Dzięki niemu zobaczysz, które pliki były modyfikowane, ale nie zostały jeszcze zapisane poleceniem commit.

  5. git clone

  6. git commit

    Tym poleceniem zapisujemy aktualny stan naszego projektu. Możemy go uzupełnić komentarzem wpisując komendę: git commit -m "komentarz".
